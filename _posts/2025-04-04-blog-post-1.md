## Java 重点内容

Java 语言简单、强大的库、健壮的检查机制、跨平台性、可移植性、动态性、半编译语言（字节码+运行时解释或JIT）。

深入学习Java需要阅读源码，深入JVM,JUC

注解+反射+设计模式一起构成了Java框架

下面是在学习中的一些重点内容：

基础语法 = 基本数据类型 + 包装类型 + 大数 + 异常 + 泛型 + 注解 + 注释规范 + 反射 + lambda表达式 + 字节流 + 序列化 + I/O + 动态代理(方法拦截与增强)  + Strem

面对对象 = 单继承 + 接口 + 多态 + 封装 + 修饰符 + 抽象类 + 内部类

重点类 = Object类 + Class类 + String类

容器 = ArrayList类 + LinkedList类 + HashMap类 + PriorityQueue类

JVM =  类加载子系统 + 运行时数据区 + 垃圾收集器 + 执行引擎(解释器、JIT、GC、本地方法接口(JNI)) + 本地方法库

运行时数据区  = 堆 + 运行时常量池 + 本地方法栈 + 程序计数器 + Java虚拟机栈 + 方法区

JRE =  JVM +  强大的库 

JDK =  JRE + 编译器 + 调试器 + 开发工具

JUC  = 多线程 + 线程池 + AQS + ConcurrentHashMap + AbstractQueuedSynchronizer + ConditionObject + CopyOnWriteArrayList + 阻塞队列源码 + 内存泄露 + 死锁 + 线程安全

## 知识点

- 对象的创建方式
- 反射的原理
- 动态代理的使用场景
- 类加载过程
  - 加载、链接（验证、准备、解析）、初始化
- 类加载器类型
  - 启动类加载器  加载核心库
  - 扩展类加载器  加载扩展库
  - 应用程序类加载器  加载用户自定义的库
  - 自定义类加载器 开发者自定义类加载逻辑
- 运行时数据区
  - 程序计数器
  - Java虚拟机栈
    - 用于存储局部变量、操作数栈、方法调用和返回等。
  - 本地方法栈 
    - 类似于JVM栈，但用于执行native方法（用C或C++编写的本地方法）。
  - 堆
    - 所有线程共享的内存区域，用于存储对象实例和数组。是垃圾回收（GC）的重点区域，分为年轻代和老年代，年轻代又分为Eden区和Survivor区。
    - (含字符串常量池)
  - 方法区
    - 所有线程共享，存储类信息、常量池、静态变量、即时编译器编译后的代码等。在HotSpot JVM中，方法区通常由永久代（PermGen）或元空间（Metaspace）实现。
    - 存储可以运行的class文件
  - 运行时常量池
    - 属于方法区的一部分，存储编译期生成的各种字面量和符号引用。
- 对象的内存布局
  - 对象头、实例数据、对齐填充
- 如何创建一个线程
